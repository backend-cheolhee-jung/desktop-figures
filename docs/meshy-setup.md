# Meshy.ai API 키 발급 & 3D 캐릭터 수동 검증 매뉴얼

이 문서는 3D 캐릭터 시스템(`feature/3d-character`)을 실제로 돌려보기 위한 준비 과정을 안내합니다.

---

## 1. Meshy.ai API 키 발급

### 1-1. 회원가입

1. https://www.meshy.ai 접속 → 우측 상단 **Sign up** (Google/이메일 가입 가능)
2. 로그인 후 대시보드 진입

### 1-2. API 키 생성

1. 좌측 하단 프로필 → **Settings** 진입
2. **API Keys** 탭 선택 (또는 직접 https://www.meshy.ai/api/keys )
3. **Create New API Key** 클릭 → 이름 입력 후 생성
4. 생성된 키(`msy_...` 형식)를 복사 — **이 화면을 벗어나면 다시 볼 수 없으니** 즉시 복사해 둘 것

### 1-3. 크레딧 확인 (중요)

- Meshy의 **OpenAPI(REST) 호출은 크레딧을 소모**합니다. 무료 가입 시 일부 크레딧이 주어지지만, 텍스트/이미지→3D 1건 + 애니메이션 2건(idle/sleep)을 합치면 캐릭터 1마리당 크레딧이 꽤 듭니다.
- 크레딧이 부족하면 생성 요청이 `402`/`429` 에러로 실패합니다 → 앱에서 캐릭터가 `failed` 상태가 됩니다.
- 대시보드 우측 상단에서 잔여 크레딧을 확인하고, 부족하면 유료 플랜을 구독하세요.

### 1-4. 테스트 모드 키 (크레딧 절약)

Meshy는 **크레딧을 쓰지 않고 파이프라인만 점검**할 수 있는 테스트 모드 키를 제공합니다:

```
msy_dummy_api_key_for_test_mode_12345678
```

- 이 키로 호출하면 **실제 생성 대신 고정된 샘플 GLB**가 빠르게 반환됩니다.
- **폴링 → 다운로드 → 렌더링** 흐름(우리 코드의 핵심)을 크레딧 없이 검증하기에 적합합니다.
- 단, 결과 모델은 입력과 무관한 더미라서 "내가 입력한 캐릭터가 나오는지"는 실제 키로 확인해야 합니다.
- 키 문자열이 바뀌었을 수 있으니, 동작하지 않으면 Meshy API 문서(https://docs.meshy.ai)의 "Test Mode" 항목에서 최신 값을 확인하세요.

> 권장 순서: **먼저 테스트 모드 키로 파이프라인 검증** → 정상 동작 확인 후 **실제 키로 최종 검증**.

## 2. .env 파일 설정

`apps/desktop/.env` 파일을 만들고 키를 넣습니다. (`.env`는 gitignore되어 커밋되지 않습니다.)

```bash
# apps/desktop/.env
VITE_MESHY_API_KEY=msy_여기에_복사한_키
```

기존에 쓰던 `VITE_AUTH_SERVER_URL` 등 다른 변수가 있다면 `.env.example`을 참고해 함께 넣어 주세요.

> ⚠️ Vite는 `VITE_` 접두사 환경변수를 **빌드 시점**에 번들에 주입합니다. `.env`를 수정하면 **dev 서버를 재시작**해야 반영됩니다.

## 3. 실행 & 수동 검증 절차

### 3-1. 앱 실행

```bash
cd apps/desktop
npm run tauri dev
```

- 앱 창이 뜨고, 캐릭터가 없으면 **Setup 페이지**가 표시됩니다.

### 3-2. 캐릭터 생성 플로우

1. Setup에서 텍스트 설명(예: "잠옷 입은 통통한 고양이") + 이름 입력 → **캐릭터 만들기**
2. 즉시 **main 페이지로 이동**하고 `"3D 캐릭터 생성 중..."` 표시되는지 확인
3. 백그라운드 폴러(15초 간격)가 동작:
   - base 모델 완료 → idle/sleep 애니메이션 2건 자동 요청
   - 둘 다 완료되면 `generation_status = ready` → **3D 캐릭터가 렌더링**됨
4. 실제 키 기준 **5~10분** 소요(테스트 모드 키는 수십 초). 그동안 앱을 켜둘 것.

**이미지 입력 검증(리스크 항목):**
- Setup에서 이미지를 드래그&드롭하여 캐릭터 생성 시, 코드가 `data:image/png;base64,...` 형식으로 Meshy `image-to-3d`에 전달합니다.
- Meshy가 data URL을 거부하고 **공개 URL만 허용**하면 여기서 `400` 에러가 납니다 → 이 경우 이미지 업로드 경로를 별도 구현해야 합니다. **이미지 입력 케이스를 꼭 한 번 테스트**해 확인해 주세요.

### 3-3. 행동(애니메이션) 생성 플로우

1. main → 📋(행동 관리) → **+** 로 행동 추가
2. 행동 이름(예: "코딩") 입력 → **생성** → `"3D 애니메이션 생성 요청 중..."` 후 detail 단계 진입
3. `🧊 3D 애니메이션 생성 중` 칩 확인 → 저장
4. ActionPanel 목록에서 상태 아이콘이 **⏳ → 🧊** 로 바뀌는지 확인(폴러가 완료 처리)
5. 해당 행동이 스케줄되어 활성화되면 main에서 그 애니메이션 GLB로 캐릭터가 바뀌는지 확인

### 3-4. 체크리스트

- [ ] dev 서버 정상 기동, Setup 표시
- [ ] 텍스트 캐릭터 생성 → pending → ready → 3D 렌더링
- [ ] 이미지 캐릭터 생성 → data URL 허용 여부 확인 ⚠️
- [ ] 행동 애니메이션 생성 → ⏳ → 🧊
- [ ] idle/active 상태 전환 시 GLB 교체 동작
- [ ] 크레딧 소진/네트워크 에러 시 failed 상태로 빠지는지

## 4. 트러블슈팅

| 증상 | 원인/조치 |
|---|---|
| 캐릭터가 계속 `생성 중`에서 안 바뀜 | dev 콘솔에서 `jobPoller error` 확인. 키 오류(401)·크레딧 부족(402/429) 가능 |
| `failed`로 즉시 전환 | API 키 오타 또는 크레딧 부족. `.env` 확인 후 dev 재시작 |
| 이미지 입력 시 400 | Meshy가 data URL 미허용 → 공개 URL 업로드 방식 필요(추가 구현) |
| 3D는 뜨는데 애니메이션 안 움직임 | GLB에 애니메이션 클립이 없을 수 있음. 콘솔에서 `animations` 길이 확인 |
| 키 변경 후에도 옛 키로 호출됨 | `.env` 저장 후 dev 서버 **재시작** 필요 |
